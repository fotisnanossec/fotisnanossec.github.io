Ping & Traceroute

Ping Analysis

# Basic connectivity ping 8.8.8.8 # Output interpretation 64 bytes from 8.8.8.8: icmp_seq=1 time=15.2 ms - Reachable, 15.2ms latency - TTL shows hop count - Packet loss percentage # Advanced ping ping -c 10 -i 0.2 target # 10 packets, 0.2s interval ping -s 1472 target # Large packet size ping6 target # IPv6

Traceroute Path Analysis

# Trace network path traceroute google.com # Output shows each hop: 1 192.168.1.1 1.2ms 2 10.0.0.1 5.3ms 3 203.0.113.1 12.1ms 4 * * * (timeout) 5 8.8.8.8 15.2ms # Continuous monitoring mtr google.com

Network Connections

Netstat Options

netstat -tuln # TCP/UDP listening netstat -an # All connections netstat -r # Routing table netstat -i # Interface stats netstat -s # Protocol stats

Modern SS Command

ss -tuln # Listening ports ss -tulpn # With process info ss -o state established # Established only ss dst 192.168.1.0/24 # Specific network

Wireshark Analysis

Capture Filters

host 192.168.1.1 port 80 tcp and port 443 not arp net 192.168.0.0/16

Display Filters

http.request.method == "GET" tcp.flags.syn == 1 dns.qry.name contains "evil" ip.addr == 192.168.1.1 tcp.analysis.retransmission

DNS Tools

Systematic Approach

Layer-by-Layer Troubleshooting

  1. 1. Physical: Cable, interface status
  2. 2. Data Link: Switch ports, VLANs
  3. 3. Network: IP config, routing
  4. 4. Transport: Port connectivity
  5. 5. Application: Service status
# Quick checks ip link show # Interface status ping gateway # Layer 3 telnet host port # Layer 4 curl -I http://site # Layer 7

SNMP Monitoring

SNMP Commands

# Get system info snmpget -v2c -c public router 1.3.6.1.2.1.1.1.0 # Walk interface table snmpwalk -v2c -c public router 1.3.6.1.2.1.2.2.1.2 # Bandwidth utilization snmpget -v2c -c public router 1.3.6.1.2.1.2.2.1.10.1

Key OIDs

1.3.6.1.2.1.1.1.0 - System description 1.3.6.1.2.1.1.3.0 - System uptime 1.3.6.1.2.1.2.2.1.10 - Interface octets in 1.3.6.1.2.1.2.2.1.16 - Interface octets out 1.3.6.1.2.1.2.2.1.8 - Interface status
