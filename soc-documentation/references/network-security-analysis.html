Network Security Analysis

SOC techniques for network threat detection and investigation

Investigating Suspicious Traffic

Initial Triage

  1. 1. Isolate traffic timeframe
  2. 2. Identify source/destination IPs
  3. 3. Examine protocols and ports
  4. 4. Check volume and patterns
  5. 5. Correlate with threat intel
# Wireshark Analysis wireshark -Y "ip.src == suspicious_ip" wireshark -Y "tcp.port == 443 and tls.handshake.type == 1" # Flow Analysis nfcapd -r flows.dat | grep suspicious_ip silk-rwfilter --sip=malicious_ip # Log Correlation grep "suspicious_ip" /var/log/suricata/eve.json

Network Compromise Indicators

Traffic Anomalies

  • • Unusual outbound connections
  • • High-volume data transfers
  • • Off-hours network activity
  • • Geographic anomalies
  • • Protocol misuse

Communication Patterns

  • • Beaconing behavior
  • • Command & control traffic
  • • Lateral movement
  • • Tunneling protocols
  • • Encrypted channels

Infrastructure Signs

  • • Rogue devices
  • • Unauthorized services
  • • DNS anomalies
  • • Certificate irregularities
  • • Port scanning activity

Data Exfiltration Detection

Detection Methods

# Volume Analysis netstat -i | awk '{print $10}' # Bytes out sar -n DEV 1 10 # Interface statistics # DNS Tunneling dig +short evil.domain.com TXT wireshark -Y "dns.qry.name contains long_string" # ICMP Tunneling tcpdump -i eth0 icmp and greater 64 # Steganography Detection binwalk suspicious_image.jpg

Exfiltration Vectors

  • • HTTP/HTTPS: POST requests, file uploads
  • • DNS: Tunneling, large TXT records
  • • Email: Attachments, cloud services
  • • FTP/SFTP: File transfers
  • • Cloud Storage: Dropbox, OneDrive sync
  • • Messaging: Slack, Teams uploads

Network-Based Threat Hunting

Hunting Hypotheses

  • • Attacker using DNS for C2
  • • Data staging before exfiltration
  • • Living-off-the-land techniques
  • • Supply chain compromises
  • • Insider threat activity

Data Sources

  • • NetFlow/sFlow records
  • • DNS query logs
  • • Proxy logs
  • • Firewall logs
  • • IDS/IPS alerts
# Hunting Queries # Long DNS queries (potential tunneling) cat dns.log | awk 'length($6) > 50 {print $0}' # Beaconing detection grep -E "GET.*/(config|update|status)" proxy.log | sort | uniq -c # High-frequency connections netstat | awk '{print $5}' | sort | uniq -c | sort -nr

Analyzing Malicious Communications

C2 Analysis

# HTTP C2 Analysis wireshark -Y "http.request.uri contains beacon" grep "User-Agent.*Bot" access.log # TLS Analysis tls_analyzer.py capture.pcap openssl s_client -connect malicious.com:443 # Traffic Frequency tcpdump -ttt -r capture.pcap | \ awk '{print $1}' | sort -n | \ uniq -c | sort -nr

Payload Analysis

  • • Extract embedded files
  • • Decode base64 content
  • • Analyze encryption patterns
  • • Identify staging servers
  • • Map infrastructure

Investigation Workflow

Phase 1: Detection

  1. 1. Alert triage and validation
  2. 2. Initial traffic analysis
  3. 3. Threat intel correlation
  4. 4. Scope determination

Phase 2: Investigation

  1. 1. Deep packet analysis
  2. 2. Flow correlation
  3. 3. Timeline reconstruction
  4. 4. Attribution analysis
