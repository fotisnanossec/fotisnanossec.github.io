Purpose

The Windows Registry is a centralized, hierarchical database storing system configuration, application settings, and user preferences. Critical for malware persistence analysis and system investigation.

Storage Architecture

# Registry Hive File Locations C:\Windows\System32\Config\ ‚îú‚îÄ‚îÄ SYSTEM ‚Üí HKEY_LOCAL_MACHINE\SYSTEM ‚îú‚îÄ‚îÄ SOFTWARE ‚Üí HKEY_LOCAL_MACHINE\SOFTWARE ‚îú‚îÄ‚îÄ SECURITY ‚Üí HKEY_LOCAL_MACHINE\SECURITY ‚îú‚îÄ‚îÄ SAM ‚Üí HKEY_LOCAL_MACHINE\SAM ‚îî‚îÄ‚îÄ DEFAULT ‚Üí HKEY_USERS\.DEFAULT # User-specific hives C:\Users\[username]\NTUSER.DAT ‚Üí HKEY_CURRENT_USER

Key Investigation Commands

# Registry Editor (GUI) regedit # Command line registry queries reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" reg query "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" # Export registry key for analysis reg export "HKEY_LOCAL_MACHINE\SOFTWARE" C:\analysis\software.reg # Search for specific values reg query HKLM /s /f "malware_name"

SOC Investigation Applications

  • Malware Persistence: Check Run/RunOnce keys for unauthorized startup programs
  • System Configuration: Analyze security policy changes and system modifications
  • User Activity: Investigate recently accessed files and application usage
  • Network Configuration: Examine proxy settings and network adapter configurations

Security Considerations

Registry modifications require administrative privileges. Always backup registry keys before making changes. Monitor registry access patterns during incident response for anomaly detection.

‚öôÔ∏è Process & Service Analysis

Process

Instance of a running program with its own memory space and execution context. Can be user-initiated or system-started with varying privilege levels.

Service

Background process running without user interface, typically starting at boot. Provides system functionality and often runs with elevated privileges.

Process Investigation Commands

# Basic process listing tasklist tasklist /svc # Show services for each process tasklist /v # Verbose output with session info # Filter processes tasklist /fi "imagename eq chrome.exe" tasklist /fi "pid eq 1234" tasklist /fi "memusage gt 100000" # High memory usage # PowerShell process analysis Get-Process Get-Process -Name "suspicious_process" Get-Process | Sort-Object CPU -Descending Get-Process | Where-Object {$_.ProcessName -match "malware"}

Service Management Commands

# Service enumeration sc query # List all services sc query state= all # All services (running and stopped) net start # Running services only # Service details sc qc "service_name" # Service configuration sc query "service_name" # Service status sc queryex "service_name" # Extended service information # Service control sc start "service_name" sc stop "service_name" sc config "service_name" start= disabled

Threat Hunting Applications

Suspicious Indicators

  • ‚Ä¢ Processes running from unusual locations (temp directories, user folders)
  • ‚Ä¢ High CPU/memory usage from unknown processes
  • ‚Ä¢ Processes with misspelled legitimate names (svchost.exe vs svch0st.exe)
  • ‚Ä¢ Services with suspicious descriptions or unexpected startup types
  • ‚Ä¢ Multiple instances of processes that should be unique

üîí Access Control & UAC

User Account Control (UAC)

Security feature preventing unauthorized system changes by requiring explicit permission or administrator credentials for administrative actions. Key defense against privilege escalation attacks.

UAC Operation Flow

  1. Application Request: Program attempts administrative action
  2. UAC Evaluation: System determines elevation requirement
  3. Secure Desktop: Screen dims, UAC prompt displayed
  4. User Decision: Approve or deny elevation request
  5. Execution: Program runs with appropriate privileges

UAC Configuration Commands

# Check UAC status reg query HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System /v EnableLUA # UAC configuration GUI UserAccountControlSettings.exe # Check if running elevated (PowerShell) ([Security.Principal.WindowsPrincipal] [Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole] "Administrator") # Registry UAC values EnableLUA = 1 # UAC enabled ConsentPromptBehaviorAdmin = 2 # Default prompt behavior PromptOnSecureDesktop = 1 # Secure desktop enabled

Security Assessment

Security Benefits

  • ‚Ä¢ Prevents unauthorized system changes
  • ‚Ä¢ Reduces malware impact through privilege limitation
  • ‚Ä¢ Creates audit trail for administrative actions
  • ‚Ä¢ Requires explicit user consent for elevation

Attack Vectors

  • ‚Ä¢ UAC bypass techniques using auto-elevation
  • ‚Ä¢ DLL hijacking for privilege escalation
  • ‚Ä¢ Token manipulation attacks
  • ‚Ä¢ Social engineering to approve prompts

Investigation Workflow

  1. 1. System Identification: OS version, hostname, domain membership
  2. 2. User Context: Current user privileges and group memberships
  3. 3. Network Configuration: IP settings, active connections, routing
  4. 4. Process Analysis: Running processes and associated network connections
  5. 5. Security Assessment: User accounts, group memberships, security policies
