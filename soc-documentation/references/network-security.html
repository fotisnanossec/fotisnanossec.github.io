Network Security

Network defense technologies and security controls

Firewalls

How Firewalls Work

Control network traffic based on predefined security rules, filtering packets between trusted and untrusted networks.

# Firewall Rules Example iptables -A INPUT -p tcp --dport 22 -j ACCEPT iptables -A INPUT -p tcp --dport 80 -j ACCEPT iptables -A INPUT -p tcp --dport 443 -j ACCEPT iptables -A INPUT -j DROP # Rule Components Source IP/Network Destination IP/Network Protocol (TCP/UDP/ICMP) Port Numbers Action (ALLOW/DENY/LOG)

Stateful vs Stateless

Stateful

  • • Tracks connection state
  • • Remembers established sessions
  • • Dynamic rule application
  • • Better security

Stateless

  • • Examines each packet individually
  • • No session awareness
  • • Static rules only
  • • Faster processing

IDS vs IPS

IDS - Detection

  • • Monitors and alerts on threats
  • • Passive detection (out-of-band)
  • • Cannot stop attacks in real-time
  • • Detailed forensic analysis
# Detection Methods Signature-based: Known attack patterns Anomaly-based: Behavioral analysis Protocol-based: Protocol violations Heuristic: Statistical analysis # Common IDS Snort, Suricata, OSSEC

IPS - Prevention

  • • Detects and blocks threats
  • • Active prevention (inline)
  • • Real-time attack stopping
  • • Can impact network performance
# Prevention Actions DROP packet BLOCK source IP RESET connection RATE LIMIT traffic QUARANTINE host # Deployment Inline with traffic flow Single point of failure risk

VPN Security

How VPNs Work

  1. 1. Create encrypted tunnel
  2. 2. Authenticate endpoints
  3. 3. Encapsulate traffic
  4. 4. Route through tunnel
  5. 5. Decrypt at destination

VPN Types

Site-to-Site: Connects networks
Remote Access: Individual users
# VPN Protocols IPSec - Network layer, strong encryption OpenVPN - SSL/TLS based, flexible WireGuard - Modern, lightweight L2TP/IPSec - Layer 2 tunneling PPTP - Legacy, weak security # Security Features Encryption (AES-256) Authentication (certificates, PSK) Integrity (HMAC) Perfect Forward Secrecy

Network Segmentation & DMZ

Network Segmentation

Divides network into isolated segments to limit attack spread and control access.

  • • Limits lateral movement
  • • Contains security breaches
  • • Improves compliance
  • • Enables micro-segmentation
# Segmentation Methods VLANs - Layer 2 separation Subnets - Layer 3 separation Firewalls - Policy enforcement ACLs - Access control lists SDN - Software-defined networking

DMZ (Demilitarized Zone)

Buffer network between internal and external networks for public-facing services.

# DMZ Architecture Internet | Firewall (External) | DMZ (Web servers, Email, DNS) | Firewall (Internal) | Internal Network # DMZ Services Web servers Email servers DNS servers FTP servers Proxy servers

Wireless Security

Security Implications

  • • Radio waves can be intercepted
  • • Physical security boundaries unclear
  • • Easy to deploy rogue access points
  • • Vulnerable to jamming attacks
  • • Client device vulnerabilities

WPA Evolution

WEP: Broken, RC4 encryption
WPA: TKIP, better than WEP
WPA2: AES-CCMP, strong security
WPA3: SAE, enhanced protection

Access Control & Zero Trust

NAC (Network Access Control)

Controls device access to network based on identity and policy compliance.

# NAC Process 1. Device connects 2. Identity verification 3. Health assessment 4. Policy evaluation 5. Access granted/denied 6. Continuous monitoring # NAC Components Authentication server Policy server Enforcement points Quarantine network

Zero Trust

"Never trust, always verify" - assumes no implicit trust.

  • • Verify every user and device
  • • Least privilege access
  • • Micro-segmentation
  • • Continuous monitoring
  • • Strong authentication
  • • Encrypted communications
