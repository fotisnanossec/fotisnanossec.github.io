HTTP & HTTPS

HTTP Components

# HTTP Request Structure GET /index.html HTTP/1.1 Host: example.com User-Agent: Mozilla/5.0 Accept: text/html Connection: keep-alive # HTTP Response Structure HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1234 Set-Cookie: session=abc123 # Methods GET - Retrieve data POST - Submit data PUT - Update resource DELETE - Remove resource HEAD - Headers only

HTTP vs HTTPS

HTTP (Port 80)

  • • Plaintext transmission
  • • No encryption
  • • Vulnerable to interception

HTTPS (Port 443)

  • • TLS/SSL encryption
  • • Certificate validation
  • • Data integrity protection

SSL/TLS Security

TLS Handshake Process

  1. 1. Client Hello (cipher suites, random)
  2. 2. Server Hello (chosen cipher, certificate)
  3. 3. Certificate verification
  4. 4. Key exchange (pre-master secret)
  5. 5. Session keys generated
  6. 6. Finished messages

Security Features

  • • Symmetric encryption (AES)
  • • Asymmetric key exchange (RSA/ECDH)
  • • Message authentication (HMAC)
  • • Certificate-based identity
  • • Perfect forward secrecy
# TLS Analysis Commands openssl s_client -connect example.com:443 nmap --script ssl-enum-ciphers -p 443 target wireshark -Y "tls.handshake.type"

DNS Resolution

DNS Resolution Steps

  1. 1. Check local cache
  2. 2. Query recursive resolver
  3. 3. Query root nameservers
  4. 4. Query TLD nameservers (.com)
  5. 5. Query authoritative nameservers
  6. 6. Return IP address
  7. 7. Cache result with TTL
# DNS Commands nslookup example.com dig example.com dig @8.8.8.8 example.com MX host -t AAAA example.com

DNS Record Types

# Common DNS Records A - IPv4 address AAAA - IPv6 address CNAME - Canonical name (alias) MX - Mail exchange NS - Name server PTR - Reverse lookup TXT - Text data SOA - Start of authority SRV - Service record # Examples example.com. A 192.168.1.100 www CNAME example.com example.com. MX 10 mail.example.com

DHCP Process

DORA Process

Discover: Client broadcasts request
Offer: Server offers IP configuration
Request: Client requests specific offer
Acknowledge: Server confirms assignment

DHCP Options

# Common DHCP Options Option 1 - Subnet mask Option 3 - Default gateway Option 6 - DNS servers Option 15 - Domain name Option 51 - Lease time Option 66 - TFTP server Option 67 - Boot filename # Lease Management ipconfig /release ipconfig /renew dhclient -r dhclient

Email Protocols

SMTP (Port 25/587)

Simple Mail Transfer Protocol - sends email

# SMTP Process 1. HELO/EHLO 2. MAIL FROM: sender 3. RCPT TO: recipient 4. DATA 5. Message content 6. QUIT # SMTP Commands telnet mail.server.com 25 EHLO client.domain.com MAIL FROM:

POP3 (Port 110/995)

Post Office Protocol - downloads email

  • • Downloads messages locally
  • • Deletes from server
  • • Single device access
  • • Offline reading
  • • Limited server storage

IMAP (Port 143/993)

Internet Message Access Protocol

  • • Syncs across devices
  • • Server-side storage
  • • Folder management
  • • Partial downloads
  • • Multi-device access

File Transfer Protocols

FTP (Ports 20/21)

Security Limitations

  • • Plaintext credentials
  • • Unencrypted data transfer
  • • Passive/active mode confusion
  • • Firewall complications
# FTP Commands ftp ftp.example.com USER username PASS password PASV LIST RETR filename STOR filename

Secure Alternatives

SFTP (Port 22)

SSH-based file transfer

FTPS (Port 990)

FTP over SSL/TLS

SCP

Secure copy over SSH

Network Management & Directory

SNMP (Port 161/162)

Simple Network Management Protocol

# SNMP Operations GET - Retrieve specific value GETNEXT - Retrieve next OID GETBULK - Retrieve multiple values SET - Modify value TRAP - Unsolicited notification # SNMP Commands snmpwalk -v2c -c public router 1.3.6.1.2.1.1 snmpget -v2c -c public router 1.3.6.1.2.1.1.1.0

Versions: v1 (plaintext), v2c (community), v3 (encrypted)

LDAP (Port 389/636)

Lightweight Directory Access Protocol

# LDAP Structure dc=example,dc=com ou=People cn=John Doe ou=Groups cn=Administrators # LDAP Operations BIND - Authenticate SEARCH - Query directory ADD - Create entry MODIFY - Update entry DELETE - Remove entry

Secure: LDAPS (port 636) with SSL/TLS

Remote Access Protocols

Telnet (Port 23)

Security Issues

  • • Plaintext transmission
  • • No encryption
  • • Credentials visible
  • • Session hijacking risk
# Telnet Usage (Legacy) telnet server.example.com 23 telnet 192.168.1.1 80 # Port testing

SSH (Port 22)

Secure Shell - encrypted remote access

# SSH Features - Encrypted communication - Public key authentication - Port forwarding - File transfer (SFTP/SCP) - Tunneling capabilities # SSH Commands ssh user@server.com ssh -i private_key user@server ssh -L 8080:localhost:80 user@server scp file.txt user@server:/path/

Network Utilities

NTP (Port 123)

Network Time Protocol - synchronizes clocks

# NTP Hierarchy (Stratum) Stratum 0 - Atomic clocks, GPS Stratum 1 - Primary servers Stratum 2 - Secondary servers Stratum 3+ - Client systems # NTP Commands ntpdate pool.ntp.org ntpq -p chrony sources timedatectl status

Importance: Log correlation, digital forensics, compliance

ICMP

Internet Control Message Protocol

# ICMP Message Types 0 - Echo Reply (ping response) 3 - Destination Unreachable 8 - Echo Request (ping) 11 - Time Exceeded (traceroute) 30 - Traceroute # Tools Using ICMP ping 8.8.8.8 traceroute google.com mtr google.com pathping google.com
