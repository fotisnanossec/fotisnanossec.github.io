Key Activities

Incident Classification & Priority

High Priority

  • • Data breach with PII
  • • Critical system compromise
  • • Active data exfiltration
  • • Ransomware deployment
  • • APT indicators

Medium Priority

  • • Malware detection
  • • Unauthorized access
  • • Policy violations
  • • Phishing campaigns
  • • DDoS attempts

Low Priority

  • • Failed login attempts
  • • Port scans
  • • Spam emails
  • • Minor policy violations
  • • False positives

Classification Factors:

# Impact Assessment - Data sensitivity (PII, PHI, financial) - System criticality (production, DR) - Business impact (revenue, operations) - Regulatory requirements (GDPR, HIPAA) # Urgency Factors - Active exploitation - Lateral movement - Data at risk - Public exposure

Initial Response Steps

First Steps

  1. 1. Stay Calm - Don't panic or rush
  2. 2. Document Everything - Time, actions, observations
  3. 3. Assess Scope - What systems affected?
  4. 4. Preserve Evidence - Don't contaminate
  5. 5. Notify IR Team - Escalate appropriately
  6. 6. Begin Containment - Stop the bleeding

Critical Questions

# Initial Assessment - What type of incident? - When did it start? - How was it discovered? - What systems affected? - Any data involved? - Is it still active? # Immediate Actions - Can we contain it? - Who needs to know? - What evidence exists? - Legal implications? - Media concerns?

Containment Strategies

Short-term Containment

  • • Network isolation
  • • Disable user accounts
  • • Block malicious IPs
  • • Quarantine systems
  • • Stop malicious processes
# Network Isolation iptables -A INPUT -s malicious_ip -j DROP netsh advfirewall firewall add rule ... # Account Management net user username /active:no passwd -l username # Process Management taskkill /f /im malware.exe kill -9 PID

Long-term Containment

  • • Patch vulnerabilities
  • • Deploy security updates
  • • Harden configurations
  • • Implement additional monitoring
  • • Update security controls
Goal: Prevent reoccurrence while maintaining business operations

Digital Evidence Preservation

Evidence Types

  • • Disk images
  • • Memory dumps
  • • Network captures
  • • Log files
  • • System snapshots
  • • Database records

Chain of Custody

  • • Who collected it?
  • • When was it collected?
  • • How was it collected?
  • • Where is it stored?
  • • Who has access?
  • • Any modifications?
# Evidence Collection dd if=/dev/sda of=/evidence/disk.img bs=4096 conv=noerror,sync memdump -p PID -o memory.dump tcpdump -i eth0 -w capture.pcap # Hash Verification md5sum disk.img > disk.img.md5 sha256sum memory.dump > memory.dump.sha256 # Timestamps date; command; date

Forensic Analysis

When Needed

  • • Criminal activity suspected
  • • Legal action anticipated
  • • Regulatory requirements
  • • Insurance claims
  • • Root cause analysis
  • • Attribution needed

Analysis Types

# Network Forensics - Packet analysis - Flow analysis - Protocol reconstruction - Communication patterns # Host Forensics - File system analysis - Registry examination - Process analysis - Timeline reconstruction # Memory Forensics - Process dumps - Network connections - Encryption keys - Malware artifacts

Recovery vs Lessons Learned

Recovery Phase
