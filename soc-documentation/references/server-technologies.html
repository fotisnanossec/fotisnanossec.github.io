Web Servers

How Web Servers Work

  1. 1. Client requests resource via HTTP
  2. 2. Server processes request
  3. 3. Retrieves/generates content
  4. 4. Returns HTTP response
  5. 5. Client renders content

Apache vs Nginx

# Apache - Process-based architecture - .htaccess configuration - Module system - Better for dynamic content # Nginx - Event-driven architecture - Better performance/concurrency - Reverse proxy capabilities - Better for static content

DNS Infrastructure

DNS Resolution

  1. 1. Query local cache
  2. 2. Contact recursive resolver
  3. 3. Query root servers
  4. 4. Query TLD servers
  5. 5. Query authoritative servers
  6. 6. Return IP address

DNS Records

A - IPv4 address AAAA - IPv6 address CNAME - Canonical name MX - Mail exchange NS - Name server TXT - Text data PTR - Reverse lookup SOA - Start of authority
