⚙️ Process Management & Monitoring

Purpose

Linux process management is critical for SOC analysts to identify suspicious processes, analyze system performance, and detect potential security threats through process behavior analysis.

Essential Process Commands

# Process listing and analysis ps aux # All processes with detailed info ps aux | grep [process_name] # Find specific process ps -ef # Full format listing ps -e --forest # Process tree view # Real-time process monitoring top # Interactive process viewer htop # Enhanced process viewer (if installed) top -p [PID] # Monitor specific process # Process control kill [PID] # Terminate process gracefully kill -9 [PID] # Force kill process killall [process_name] # Kill all instances of process pkill -f [pattern] # Kill processes matching pattern # Background process management jobs # List active jobs nohup [command] & # Run process in background disown # Remove job from shell's job table

Daemon vs Regular Process

Daemon Process

  • • Runs in background without terminal
  • • Started at boot or by system
  • • Parent PID typically 1 (systemd/init)
  • • Examples: sshd, httpd, mysqld

Regular Process

  • • User-initiated from terminal
  • • Has controlling terminal
  • • Child of shell process
  • • Examples: vim, grep, ls

SOC Investigation Applications

  • Malware Detection: Identify processes with unusual names or resource consumption
  • Persistence Analysis: Check for unauthorized daemon processes
  • Performance Investigation: Analyze CPU and memory usage patterns
  • Network Activity: Correlate processes with network connections

Threat Hunting Indicators

Monitor for processes running from /tmp or /var/tmp, unusual parent-child relationships, processes with hidden names (starting with dots), and unexpected network connections.

📁 File System & Permissions

Linux Directory Structure

# Essential directory hierarchy / # Root directory ├── /etc # System configuration files ├── /home # User home directories ├── /var # Variable data (logs, mail, tmp) │ ├── /var/log # System and application logs │ └── /var/tmp # Temporary files (persistent across reboots) ├── /usr # User programs and data ├── /bin # Essential command binaries ├── /sbin # System administration binaries ├── /tmp # Temporary files (cleared on reboot) ├── /proc # Process and kernel information └── /sys # Kernel and system information

File Permissions System

Permission Structure

Linux uses a 3-bit permission system for User (owner), Group, and Other (world):

# Permission bits: rwx rwx rwx # User permissions | Group permissions | Other permissions # 4 (read) + 2 (write) + 1 (execute) = 7 (full permissions) # Common permission examples: 755 = rwxr-xr-x # Owner: full, Group/Other: read+execute 644 = rw-r--r-- # Owner: read+write, Group/Other: read only 700 = rwx------ # Owner: full, Group/Other: no access

File Permission Commands

# View file permissions ls -la # Detailed listing with permissions ls -la /etc/passwd # Check specific file permissions stat filename # Detailed file information # Modify permissions chmod 755 filename # Set specific permissions chmod u+x filename # Add execute for user chmod g-w filename # Remove write for group chmod o-r filename # Remove read for others # Change ownership chown user:group filename # Change owner and group chown user filename # Change owner only chgrp group filename # Change group only # Special permissions chmod +s filename # Set SUID/SGID bit chmod +t directory # Set sticky bit (for directories)

File Search & Discovery

# Find command variations find / -name "filename" # Search by name find /home -type f -name "*.log" # Find log files find /var -size +100M # Files larger than 100MB find / -perm 4755 # Find SUID files find /tmp -mtime -1 # Modified in last 24 hours # Search file contents grep -r "pattern" /var/log/ # Search in all log files grep -i "error" /var/log/syslog # Case-insensitive search zgrep "pattern" /var/log/*.gz # Search compressed logs # File system analysis du -sh /var/log/ # Directory size df -h # Disk space usage lsof +D /var/log/ # Open files in directory

Links and File References

Hard Links

  • • Direct reference to inode
  • • Same file system only
  • • File persists if original deleted
  • • ln file hardlink

Symbolic Links

  • • Pointer to file path
  • • Can cross file systems
  • • Breaks if target deleted
  • • ln -s file symlink

Security Implications

Monitor SUID/SGID files for privilege escalation, check /tmp and /var/tmp for suspicious files, and investigate unusual file permissions in system directories. Pay attention to hidden files (starting with .).

🔐 System Administration & Access Control

User Privilege Management

su (Switch User)

  • • Switches to another user account
  • • Requires target user's password
  • • Creates new shell session
  • • Full user environment switch

sudo (Superuser Do)

  • • Executes commands as another user
  • • Uses current user's password
  • • Granular permission control
  • • Comprehensive audit logging
# su command usage su # Switch to root (requires root password) su - username # Switch to user with environment su -c "command" username # Execute command as user # sudo command usage sudo command # Execute command as root sudo -u username command # Execute command as specific user sudo -l # List allowed sudo commands sudo -i # Interactive root shell # sudo configuration sudo visudo # Edit sudoers file safely sudo cat /etc/sudoers # View sudo configuration sudo tail -f /var/log/auth.log # Monitor sudo usage

Automated Task Management (Cron)

Cron System

Cron daemon (crond) executes scheduled tasks automatically. Critical for system maintenance and potential attack vector for persistent malware execution.

# Cron management commands crontab -l # List current user's cron jobs crontab -e # Edit current user's cron jobs crontab -u username -l # List specific user's cron jobs sudo crontab -l # List root's cron jobs # System-wide cron files cat /etc/crontab # System crontab ls /etc/cron.d/ # Additional cron files ls /etc/cron.daily/ # Daily cron jobs ls /etc/cron.hourly/ # Hourly cron jobs # Cron syntax: minute hour day month weekday command # Example: 0 2 * * * /usr/bin/backup.sh # Daily at 2 AM

Security Monitoring

Regularly audit cron jobs for unauthorized tasks, monitor sudo logs for privilege escalation attempts, and verify package integrity for signs of compromise. Check for unusual scheduled tasks in all cron directories.

📊 Log Analysis & System Monitoring

System Log Locations

# Primary log directory: /var/log/ ├── /var/log/syslog # General system messages (Debian/Ubuntu) ├── /var/log/messages # General system messages (RedHat/CentOS) ├── /var/log/auth.log # Authentication logs (Debian/Ubuntu) ├── /var/log/secure # Authentication logs (RedHat/CentOS) ├── /var/log/kern.log # Kernel messages ├── /var/log/dmesg # Boot and hardware messages ├── /var/log/cron # Cron job execution logs ├── /var/log/mail.log # Mail server logs ├── /var/log/apache2/ # Apache web server logs ├── /var/log/nginx/ # Nginx web server logs └── /var/log/audit/ # SELinux/AppArmor audit logs

Log Analysis Commands

# Real-time log monitoring tail -f /var/log/syslog # Follow log in real-time tail -n 100 /var/log/auth.log # Show last 100 lines multitail /var/log/syslog /var/log/auth.log # Monitor multiple logs # Log search and analysis grep "Failed password" /var/log/auth.log # Find failed login attempts grep -i "error" /var/log/syslog # Case-insensitive error search zgrep "pattern" /var/log/*.gz # Search compressed logs # Log rotation and management ls -la /var/log/*.log # Current active logs ls -la /var/log/*.gz # Rotated/compressed logs logrotate -d /etc/logrotate.conf # Test log rotation config

System Resource Monitoring

# Disk space analysis df -h # Human-readable disk usage du -sh /var/log/ # Directory size summary du -ah /var/log/ | sort -rh # Sort by size, largest first # Memory and CPU monitoring free -h # Memory usage (human-readable) cat /proc/meminfo # Detailed memory information cat /proc/cpuinfo # CPU information uptime # System load and uptime # Process and system monitoring vmstat 1 # Virtual memory statistics iostat 1 # I/O statistics sar -u 1 10 # CPU utilization (10 samples)

SOC Log Analysis Workflow

  1. Baseline Establishment: Understand normal log patterns and volumes
  2. Authentication Analysis: Review login attempts and privilege escalations
  3. Error Investigation: Analyze system errors and application failures
  4. Timeline Construction: Correlate events across multiple log sources
  5. Anomaly Detection: Identify unusual patterns or unexpected events

Critical Log Events

  • • Failed authentication attempts (brute force indicators)
  • • Privilege escalation events (sudo usage, su attempts)
  • • New user account creation or modification
  • • Unusual network connections or service starts
  • • System configuration changes or package installations
