Cross-Platform Operating Systems

SOC Analyst Essentials

Essential knowledge for SOC analysts working in mixed Windows/Linux environments. Understanding both platforms enables effective monitoring, incident response, and security analysis across diverse infrastructures.

Security Models Comparison

Windows Security Model

# Access Control Lists (ACLs) icacls C:\sensitive-data # Security Identifiers (SIDs) whoami /user # User Account Control (UAC) # Privilege elevation prompts

Key Concepts:

  • ACLs: Discretionary Access Control Lists
  • Security Tokens: Contain user/group SIDs
  • NTFS Permissions: File system level security

Linux Security Model

# Permission bits (rwx) ls -la /etc/passwd # Access Control Lists (extended) getfacl /sensitive-data/ setfacl -m u:username:rwx /path/ # SELinux/AppArmor sestatus aa-status

Key Concepts:

  • Permission Bits: Owner, Group, Other (rwx)
  • UID/GID: User and Group identifiers
  • sudo/su: Privilege escalation

SOC Analyst Applications

Log Analysis Across Platforms

# Windows Event Logs Get-WinEvent -LogName Security | Where-Object {$_.Id -eq 4624} # Linux System Logs grep "Failed password" /var/log/auth.log journalctl -u sshd --since "2024-01-01"

Incident Response

# Windows IR commands Get-Process | Export-Csv processes.csv netstat -ano > connections.txt # Linux IR commands ps aux > processes.txt netstat -tuln > connections.txt

Platform-Specific Threat Indicators

Windows IoCs:

  • Unusual PowerShell execution
  • Suspicious registry modifications
  • Anomalous service installations

Linux IoCs:

  • Suspicious cron jobs
  • Hidden files/directories
  • Modified system binaries

Quick Reference Commands

System Information

# Linux uname -a # System information ps aux # Running processes # Windows systeminfo # System information Get-Process # Running processes

Network Commands

# Linux ip addr show # Network interfaces ss -tuln # Network connections # Windows ipconfig /all # Network configuration netstat -ano # Network connections
